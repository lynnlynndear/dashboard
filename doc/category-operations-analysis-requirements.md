# 品类运营经营分析系统需求文档

版本：V1 需求校准稿  
日期：2026-05-28  
参考：`ref/ref.mov` 演示系统呈现方式、`doc/2026BP.xlsx` 年度 BP 模板

## 1. 系统定位

### 1.1 产品名称

品类运营经营分析系统

### 1.2 第一用户

经营分析岗。

系统不是给老板日常点开的普通财务看板，也不是销售审批系统。它面向经营分析岗在经营会前完成数据整理、经营诊断、问题归因、动作建议和汇报输出。

### 1.3 核心场景

经营会工具。

系统需要把财务口径、BP 目标、ERP/电商/门店/订单明细、业务分摊规则和新品 Roadmap 整合成可解释的经营结论，帮助经营分析岗回答：

- 哪些品类、SPU、SKU、渠道、客户类型组合真正赚钱？
- 哪些组合只是流水大但经营贡献差？
- 哪些营销投入有效，哪些只是在买规模？
- 新品没有达成 BP 是因为缺货、预热不足、上新节奏问题、渠道问题、投放问题，还是产品需求弱？
- 下月品类、渠道、营销投入应该加码、维持、优化、收缩还是观察？

### 1.4 V1 交付形态

本地可跑系统 + 标准 Excel 模板。

V1 不做云端多人协作、不做账号权限体系、不做 BP 审批流、不做 ERP/API 自动同步。先跑通本地导入、计算、诊断和经营会输出。

## 2. 参考视频呈现方式

系统前端参考 `ref/ref.mov` 中的一页式经营决策看板呈现方式：

- 顶部固定蓝色系统标题区，展示系统名和一句经营分析定位。
- 顶部操作按钮包含：导入 Excel、下载模板、导出模型、导出汇报版、打印/PDF。
- 横向模块导航，支持在经营驾驶舱、BP、Roadmap、数据导入、参数设置、口径说明、产品利润、渠道盈亏、营销敏感性、成本敏感性、经营会报告之间切换。
- 首页不是普通财务报表，而是经营结论页：关键指标、利润桥、风险/机会原因、下月动作、决策入口。
- 每个模块需要把数字翻译成经营语言，不能只展示表格。
- 输出建议需要有依据，例如“销售未达 BP，但主要由缺货导致”“销售增长但跑输大盘，投放策略需要优化”。

## 3. V1 范围

### 3.1 必做范围

1. 年度 BP 上传与锁定。
2. 标准 Excel 模板导入。
3. 品类/SPU/SKU/渠道/客户类型/月度经营数据清洗与汇总。
4. 新品/SPU Roadmap 分页。
5. 产品/品类利润分析。
6. 渠道盈亏分析。
7. 营销投入敏感性分析。
8. BP 偏差分析与新品偏差归因。
9. 经营会结论页与汇报导出。

### 3.2 V1 暂不做

1. BP 审批流。
2. 云端权限、多用户协同。
3. 自动对接 ERP、电商平台、门店系统 API。
4. 日常订单审批流。
5. 复杂机器学习归因模型。
6. 财务总账系统替代。

## 4. 数据颗粒度

V1 核心分析颗粒度固定为：

```text
月份 × 品类/系列 × SPU × SKU × 渠道 × 客户类型
```

如果某些源数据暂时无法到 SKU，则至少保留到 SPU；如果客户类型缺失，则使用默认值“未分类客户类型”，但系统需要提示数据完整度不足。

## 5. 数据来源

### 5.1 已确认数据来源

当前数据来源为混合模式：

- 财务口径输出的汇总数据。
- 业务人员提供的实际渠道分摊比例。
- 平台扣点。
- 促销归属。
- 研发项目投入。
- 门店租金分摊。
- 经销商返利。
- 样品费用。
- ERP 进销存明细。
- 电商后台明细。
- 门店系统明细。
- 订单系统导出明细。
- 手动输入的行业大盘数据。

### 5.2 V1 数据策略

V1 使用“财务口径校验 + 业务明细拆解 + 人工规则补充”的策略：

```text
财务 Excel：用于校验年度/月度总额和财务口径一致性。
ERP/电商/门店/订单明细：用于拆解到品类、SPU、SKU、渠道、客户类型。
人工规则表：用于维护费用归属、渠道分摊、产品阶段、行业大盘、特殊场景。
```

## 9. 核心经营口径

### 9.1 分层利润口径

系统需要按照以下层级拆解经营结果：

```text
GMV
- 退货/退款/折扣
= GSV / 净销售口径

净销售收入
- 产品成本/制造成本/采购成本
= 商品毛利

商品毛利
- 平台扣点
- 经销商返利
- 渠道佣金
- 门店租金分摊
- 物流履约/仓储/售后
= 渠道贡献

渠道贡献
- 促销费用
- 投流费用
- 达人/直播/活动费用
= 营销后贡献

营销后贡献
- 研发投入分摊
- 样品费用
- 品类固定费用
= 品类经营贡献
```

### 9.2 经营会核心指标

至少输出：

- GMV。
- GSV。
- 净销售收入。
- 退货率。
- 折扣率。
- 产品毛利。
- 产品毛利率。
- 渠道贡献。
- 渠道贡献率。
- 营销费用。
- 营销费用率。
- 营销后贡献。
- 品类经营贡献。
- 经营贡献率。
- 库存金额。
- 可售库存。
- 库存周转。
- BP 达成率。
- 同比增速。
- 环比增速。
- 大盘增速。
- 跑赢/跑输大盘幅度。

## 10. 产品阶段判断

### 10.1 判断方式

产品阶段由系统自动判断，并保留经营分析岗手动标记权限。

自动判断依据：

- 上市时间。
- 销量趋势。
- 复购表现。
- 库存压力。
- 经营贡献。
- 营销投入强度。
- 是否跑赢大盘。
- 是否达成 BP。

### 10.2 阶段分类

```text
新品观察期：上市 0-2 个月。
新品复盘期：上市第 3 个月。
新品验证期：上市 4-6 个月。
成长期：销售增速跑赢大盘，动销/复购改善。
成熟期：销售稳定，重点看贡献和费用效率。
清库存期：库存压力高，重点看动销和现金回收。
重审/收缩期：超过 6 个月仍亏损且规模没有改善。
```

### 10.3 新品亏损规则

```text
上市 0-2 个月：允许战略亏损，系统输出“观察”。
上市第 3 个月：必须复盘，判断继续投放、优化投放、调整价格/渠道或降低投入。
上市 4-6 个月：至少需要销售增速、复购、渠道效率、经营贡献中的一个核心指标明显改善。
超过 6 个月仍亏损：不能继续按新品保护，必须进入重审、收缩、清库存或战略保留决策。
```

### 10.4 手动修正规则

手动修改产品阶段时，需要记录：

- 原自动阶段。
- 人工调整阶段。
- 调整原因。
- 调整人。
- 生效月份。
- 是否纳入经营会说明。

## 11. 营销费用归属

### 11.1 已确认规则

营销费用归属规则：

```text
一笔营销费用优先按照推广产品归属。
达人投放算产品营销费用。
没有明确产品的营销费用，进入品类共摊池，再按品类 GMV 拆分。
如果不是针对导流某一渠道店铺的营销费用，例如达人投放，则按当月渠道 GMV 比例拆分。
```

### 11.2 归属优先级

系统归属优先级：

```text
明确 SKU + 渠道：归到 SKU × 渠道。
明确 SPU + 渠道：归到 SPU × 渠道。
明确产品但不明确渠道：归到产品，再按当月渠道 GMV 拆分。
明确渠道但不明确产品：归到渠道，再按品类 GMV 拆分。
不明确产品也不明确渠道：进入品类共摊池，按品类 GMV 拆分。
```

弱归因费用需要打标：

```text
归因类型 = 直接归因 / 渠道拆分 / 品类共摊 / 弱归因
```

## 12. 营销敏感性分析

### 12.1 分析窗口

营销效果默认观察花费当天起的三个月内：

```text
0-30 天：即时转化。
31-60 天：延迟转化。
61-90 天：长尾转化。
```

经营会可以查看当月效果，但系统结论不能只看当月。

### 12.2 指标

系统至少计算：

```text
营销投入产出比 = 净销售收入 / 营销费用
毛利拉动系数 = 商品毛利增量 / 营销费用增量
贡献拉动系数 = 经营贡献增量 / 营销费用增量
费用率 = 营销费用 / 净销售收入
边际回报变化 = 本期投入回报 - 对照期投入回报
销售敏感度 = 销售增幅 / 营销费用增幅
大盘相对增速 = 本品类/产品/渠道增速 - 行业大盘增速
BP达成率 = 实际 / BP目标
```

### 12.3 参照口径

营销效果判断不能只看单一 ROI，需要同时看：

- 环比。
- 同比。
- 过去 3 个月均值。
- 行业大盘增幅。
- BP 达成率。
- 产品阶段。
- 库存压力。
- 经营贡献变化。

### 12.4 结论类型

系统输出五类结论：

```text
加码：
销售跑赢大盘，经营贡献改善，或新品关键成长指标明显改善。

维持：
销售接近大盘，贡献可接受，费用率没有继续恶化。

优化：
销售增长但跑输大盘，或贡献下降但处于新品/大促/清库存合理场景。

观察：
新品 0-2 个月，达人投放 90 天窗口未完整，数据证据不足。

重审/收缩：
超过 6 个月仍亏损，销售不及大盘，库存压力没有缓解，费用效率恶化。
```

### 12.5 增收不增利判断

如果销售增长但经营贡献下降，系统不能直接判定为无效投放。

需要结合：

- 是否大促期。
- 是否新品上市。
- 是否清库存。
- 是否销售跑赢大盘。
- 是否 BP 达成。
- 是否库存压力缓解。
- 是否费用率恶化。
- 是否同比表现改善。

示例判断：

```text
销售增长但贡献下降，且处于大促期，销售跑赢大盘：
判定为“销售有效，利润承压”，建议优化费用率和价格结构。

销售增长但贡献下降，且销售跑输大盘：
判定为“低效增长”，建议调整投放策略。

新品销售增长但贡献为负，且上市未满 2 个月：
判定为“观察”，不直接收缩。

成熟品销售增长但贡献下降，且费用率恶化：
判定为“优化/收缩”。
```

## 14. 新品 BP 偏差归因

### 14.1 前置判断

新品实际低于 BP 时，系统先判断四个前置条件：

```text
是否按计划预热？
是否按计划到货？
是否按计划现货开卖？
是否有足够可售库存？
```

### 14.2 归因类型

新品偏差归因分为：

```text
供应问题：缺货、到货延迟、可售库存不足、断码断色、渠道未铺货。
节奏问题：种草太晚、预热期不足、上新时间晚于 BP、活动节点错过。
需求问题：曝光足够但转化差、收藏加购低、搜索/点击弱、复购信号弱。
渠道问题：某渠道未按计划上架、门店铺货慢、经销商动销差、电商流量不及预期。
投入问题：实际营销费用低于 BP、投放渠道错配、费用花了但销售跑输大盘。
BP 问题：目标过高、定价假设错误、库存/渠道/费用假设不成立。
```

### 14.3 输出示例

系统需要输出类似以下经营语言：

```text
销售未达 BP，但主要由缺货导致：
实际可售库存仅达到 BP 的 58%，断货 12 天，不应判定为需求弱。

销售未达 BP，且预热不足：
实际上新前预热仅 5 天，低于 BP 计划 21 天，首月转化判断需后移。

销售未达 BP，且投放低效：
营销费用达到 BP 的 110%，曝光充足，但销售仅达 BP 的 62%，且低于大盘增幅。

销售未达 BP，可能是 BP 假设偏高：
库存、投放、渠道铺货均达成，但销售和转化持续低于目标。
```

## 15. 产品/品类利润分析

### 15.1 模块目标

分析每个品类、SPU、SKU 的真实经营贡献，而不是只看销售额。

### 15.2 产品分类

系统按销售规模和经营贡献输出：

```text
利润明星：收入高，贡献高，优先保障库存、产能和营销资源。
规模产品：收入高，贡献一般或偏低，需要优化费用、价格和渠道结构。
潜力产品：收入低但贡献率高，可以测试加码。
拖累产品：收入低或中等，贡献为负，且无新品/清库存/战略理由。
清库存产品：库存压力高，允许阶段性牺牲利润换动销和现金回收。
战略新品：上市早期允许亏损，但必须跟踪 90 天成长信号。
```

### 15.3 输出内容

模块输出：

- 品类经营贡献排行。
- SPU/SKU 经营贡献排行。
- 销售规模 vs 贡献率矩阵。
- 产品阶段标签。
- BP 达成率。
- 同比/环比/大盘对比。
- 库存压力。
- 建议动作。

### 16.3 判断示例

```text

提示扣点、促销、返利、履约或营销费用是否过高。

渠道收入低但贡献率高：

```

## 21. 数据校验

导入后系统需要输出数据质量检查：

- 必填字段缺失。
- 月份格式异常。
- 品类/SPU/SKU 无法匹配。
- 渠道无法匹配。
- 客户类型缺失。
- 营销费用无法归属。
- BP 与实际维度不一致。
- 金额合计与财务口径不一致。
- 退货率、费用率、扣点异常。
- 可售库存缺失。
- 行业大盘数据缺失。

数据质量检查需要区分：

```text
阻断项：不修复无法计算。
警告项：可计算但结论可信度下降。
提示项：不影响计算，仅提示补充。
```
